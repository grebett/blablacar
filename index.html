What about my tech stack?

I do apply for a frontend engineer job, yet, I like to define myself as a fullstack web developer. Javascript addict, I'm curious about other languages and have practised a few. Let me present you my tech stack.

JavaScript mon amour! This language has had a very strange life. Everyone was despising it as a language just good for script kiddies. And then jQuery happened. And Node.js. And all of a sudden, JavaScript became cool! Today, you can't imagine the web without it. And if you stick to the good parts (which are more and more numerous as the new ES versions come out), JavaScript is the power tool you have to masterize.

I do like vanillaJS, and try to stick to it instead of using jQuery. Yet, I'm not a JS extremist who loves to write each piece of its own code by hand. The web today is a place to share little blocks to build bigger ones. Very interested in frameworks that endorse these vision, I've worked a lot with angularJS 1.x and, none of the less, its directives. A whole ecosystem for single page applications, but also cross-platform mobile ones!

Lately, I have discovered Polymer. Polymer is not a framework. It's a library (written by Google Chrome devs) wrapping the web components. This concept is, to me, the next revolution in web development. Imagine a container composed of an HTML template, a script and a stylesheet. This container is independant, has no effect on other containers except its children. You now have a reusable, easy to develop, little component. A small brick which just waits to be assembled with its numerous siblings!

I know, this is not the official logo of Node. But heh, design matters. I've started programming with node during my first year of school. It was not taught in any project, but I found the principle amazing. Now, Node.js has become a must have for a lot of usages, and is the basis of numeros dev tools (as you may know since you are using Grunt). JavaScript outside the browser, who saw that coming?

You may say: «Well, JavaScript is fun, but I want a real language for real backend programming.» I heard you. Why don't have a look on go language (golang) ? First of all, the gopher looks a lot like your Mascot. First good point. Then, this language is maybe one of the most mordern, easy to use and efficient language I've encountered (Geeks I know still try to make me learn Rust). Golang is the illegitimate child of Python and C. Hard typing, but interpolation. Error checking first. Concurrency. Safe pointers. Very fast compilation... I won't say more, just have a look if you haven't already or check out my github golang repositories.

In 42 school, we start learning programming with C language. Well, that's not exactly true. We don't really use the language itself, but only its core. Not a single function from the standard library allowed, we have to recode everything from scratch. Imagine using your own printf? This way of learning has its advantages, and algorithms, unix calls and functions prototypes have no more secret for us.

Lately, I've been curious about Machine Learning (especially Deep Learning). It's quite the premices so I won't say I masterize it at all. Yet, in order to use Tensor Flow (the open source Deep Learning library released by Google), I've start to learn Python. I can't wait to look forward and see how far this new field can lead us!
